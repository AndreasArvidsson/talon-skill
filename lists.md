# Talon lists

A Talon list maps spoken phrases (keys) to values.

In [spoken phrase rules](./spokenPhraseRule.md), you speak the key; in Talon script, you receive the mapped value.

Lists are commonly used for:

- keys (`air` -> `a`)
- symbols (`dash` -> `-`)
- app/domain terms

## Declaring a list

Lists are declared on a module and appear in the `user.` namespace.

```py
from talon import Module

mod = Module()
mod.list("letter", desc="Letter/alphabet key names")
```

This declares `user.letter`.

## Defining list values in `.talon-list` files

Talon lists are primarily defined in [`.talon-list`](./talonListFiles.md) files. This is static list data.

```talon
list: user.letter
-
air: a
bat: b
cap: c
```

## Defining list values in Python

List values can also be defined on a [context](./contexts.md). This is useful when values are dynamic or generated by logic.

Use `.talon-list` for declarative list data and Python when values need logic, reuse, or dynamic generation.

```py
from talon import Context

ctx = Context()

ctx.lists["user.letter"] = {
    "air": "a",
    "bat": "b",
    "cap": "c",
}
```

A list can also be set from a list of strings, which maps each item to itself.

```py
ctx.lists["user.git_command"] = ["status", "commit", "push"]
```

## Using lists in spoken rules

Reference a list with braces in a [spoken phrase rule](./spokenPhraseRule.md):

```talon
insert {user.letter}: insert(letter)
press {user.letter}: key(letter)
```

`letter` is the resolved value from the list entry.

## Context behavior

`ctx.lists[...]` sets the list definition for that context. This is a context-specific definition, not an append operation.

When multiple contexts define the same list and more than one matches, Talon does not merge those list definitions. The most specific one is used instead.
